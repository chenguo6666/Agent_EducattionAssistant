# 教育助手 AI Agent 项目简述

## 1. 项目简介

本项目是一个面向教育场景的 AI Agent Web 应用。  
用户可以在网页中直接提问，也可以上传学习资料，让系统自动识别任务类型，并根据任务内容决定是：

- 直接像聊天助手一样回复
- 调用教育工具完成摘要、出题、知识点提取、复习提纲、资料问答等任务

项目目标是展示一个完整的 AI Agent 基本流程：

- 用户输入任务
- 系统识别任务意图
- 选择是否调用工具
- 返回结果并在前端展示执行状态

## 2. 核心功能

- 用户注册、登录
- 普通自由对话
- 上传资料并进行资料分析
- 自动识别任务类型
- 调用预设工具完成教育任务
- 显示执行状态、任务意图、工具调用轨迹
- 结果导出
- 错题记录与查看

## 3. 技术栈

- 前端：Vue 3 + Vite + TypeScript
- 后端：FastAPI + SQLAlchemy + SQLite
- Agent：LangChain + Qwen（通过硅基流动 OpenAI 兼容接口接入）
- 文档处理：PDF / DOCX / TXT 文本抽取
- 检索增强：轻量 RAG

## 4. 项目亮点

- 支持“普通聊天”和“工具任务”共存，不是所有问题都强制走资料问答
- 系统会识别用户当前消息的任务类型，而不是只看整个会话是否有资料
- 前端会展示任务状态和工具调用过程，用户可以看到系统在做什么
- 结合了教育场景，适合总结、出题、复习和资料问答

## 5. 系统工作流程

1. 用户输入问题或上传资料
2. 后端先识别当前消息属于哪类任务
3. 如果是普通聊天，直接调用大模型回复
4. 如果是教育任务，调用对应工具
5. 如果涉及资料问题，先检索资料片段，再生成回答
6. 前端展示状态、意图、工具轨迹和最终结果

## 6. 预设工具

当前项目中的主要工具包括：

- `summarize_material`：总结资料
- `generate_quiz`：生成题目
- `extract_key_points`：提取知识点
- `build_study_outline`：生成复习提纲
- `retrieve_document_chunks`：检索资料片段
- `answer_with_context`：基于资料回答问题

## 7. 常见问题与简答

### Q1：这个项目解决了什么问题？
它解决的是“学生面对资料时，如何用一个智能助手自动完成总结、出题、提纲生成和资料问答”的问题。

### Q2：为什么叫 AI Agent，而不是普通聊天机器人？
因为它不是只生成一段回答，而是会先识别任务，再决定是否调用工具，并把工具执行结果整合后返回。

### Q3：系统怎么判断要不要调用工具？
系统采用混合路由：

- 明确任务先用规则快速识别
- 较模糊任务再交给大模型判断
- 只有真正需要工具时才调用工具

### Q4：如果用户只是说“你好”，系统会怎么做？
它会把这类消息识别为普通自由对话，直接走聊天回复，不调用资料工具。

### Q5：如果用户上传了资料，但下一句和资料无关怎么办？
系统只根据“当前这条消息”的意图来判断，不会因为会话里有资料就强制走资料问答。

### Q6：如果用户说“帮我看看可能考哪些题”，系统能识别吗？
可以。系统会把这类表达识别为出题类任务，而不只是依赖“出题”这两个关键词。

### Q7：项目中使用的大模型是什么？
当前项目主要接入的是 Qwen，用于聊天回复、任务分类和部分工具生成能力。

### Q8：项目里 LangChain 的作用是什么？
LangChain 用来组织 Agent 和工具调用流程，让系统具备“选择工具并执行”的能力，而不是只靠单次提示词。

### Q9：RAG 在这里的作用是什么？
当用户上传资料并提出资料相关问题时，系统会先检索相关片段，再根据片段生成更准确的回答。

### Q10：前端为什么要展示状态和工具轨迹？
这样用户可以知道系统是否正确理解了自己的需求，也能更直观看到任务是如何被处理的。

### Q11：项目目前的局限是什么？

- 当前仍然是课程项目级别的轻量实现
- 工具种类还可以继续扩展
- 复杂多轮任务的稳定性还有优化空间
- 目前使用的是临时公网分享，不是正式长期部署

### Q12：后续还能继续做什么？

- 增加更多教育工具
- 做更稳定的任务分类器
- 增强多轮对话记忆
- 做正式云部署
- 增加教师端或班级端功能

## 8. 一句话总结

这是一个面向教育场景的 AI Agent 全栈项目，能够根据用户自然语言任务，自动判断是否需要调用工具，并完成资料分析、出题、复习辅助和智能问答。
