# Sprint 1 执行计划

## 1. Sprint 目标

交付一个可演示的最小闭环系统：

- 用户能够注册和登录
- 用户能够输入教育任务
- 系统能够识别任务类型并调用 Mock Tool
- 页面能够看到状态变化和最终结果

## 2. Sprint 1 范围映射

来自产品 backlog 的 Sprint 1 条目：

- 登录与认证
- 对话交互界面
- Agent 核心逻辑
- 任务状态反馈
- 界面简洁直观

来自 Sprint 1 任务分解表的核心任务：

- 设计用户数据库表
- 开发后端注册/登录接口 + JWT
- 开发前端登录与注册页面
- 实现 Token 持久化与路由登录态管理
- 搭建主对话页面布局
- 实现消息气泡与 Markdown 渲染
- 搭建后端 LangChain 基础框架与 LLM 连接
- 定义 Agent 提示词模板
- 实现意图识别与 Mock 工具分发逻辑
- 编写摘要工具与出题工具
- 定义任务状态协议
- 实现状态提示桩
- Sprint 1 全链路测试与 Bug 修复

## 3. 开发顺序

建议不要完全按 Excel 的天数顺序写，而是按依赖顺序推进：

### 第一阶段：工程初始化

- 初始化 `frontend`
- 初始化 `backend`
- 配置基础目录结构
- 建立前后端 `.env.example`
- 建立 README 中的启动说明

### 第二阶段：后端优先打通主链路

- 建 `users` 表
- 完成注册/登录接口
- 接入 JWT
- 定义 `POST /api/chat/execute`
- 定义任务状态枚举和统一响应结构
- 实现摘要工具与出题工具
- 用简单规则或 Prompt 完成意图分发

### 第三阶段：前端建立可用界面

- 登录/注册页面
- 路由守卫
- 对话主页面
- 输入框、提交按钮、消息列表
- 状态显示区
- Markdown 渲染

### 第四阶段：联调

- 登录态验证
- 调用任务接口
- 显示意图、步骤、结果
- 处理异常状态

### 第五阶段：验收

- 手工测试主要用户路径
- 修复关键 Bug
- 准备演示数据与演示脚本

## 4. Definition of Done

Sprint 1 完成的标准：

- 未登录用户不能进入主页面
- 登录成功后可进入对话页
- 输入“总结这篇课文”能返回摘要结果
- 输入“生成 5 个选择题”能返回题目结果
- 输入综合任务时能依次调用两个工具
- 页面有明确状态提示
- 页面不会因为空输入或后端异常直接崩溃

## 5. 建议的任务拆分方式

为了更适合实际开发，建议把 Sprint 1 任务拆成下面 8 个开发包：

1. 后端项目初始化
2. 认证模块
3. Agent 基础骨架
4. Mock Tool 模块
5. 聊天接口与统一响应结构
6. 前端项目初始化与路由
7. 登录页与聊天页
8. 联调、测试与演示准备

## 6. 验收清单

每完成一个开发包，就检查以下问题：

- 接口字段名是否和文档一致
- 状态枚举是否一致
- 前端是否已处理加载中和失败态
- Mock Tool 输出是否稳定
- 综合任务是否支持顺序执行

## 7. 代码阶段的第一批实际任务

进入开发时，我建议先做下面这批：

1. 初始化 `frontend` Vue 3 + Vite + TypeScript
2. 初始化 `backend` FastAPI 项目骨架
3. 建立后端 `auth`、`chat`、`agent`、`tools` 模块
4. 定义统一响应格式和状态枚举
5. 完成前端路由与登录页骨架

这批任务完成后，整个项目会从“只有文档”变成“可启动的工程骨架”，再往下写业务会顺得多。
