# Sprint 1 演示脚本

## 1. 演示目标

在 3 到 5 分钟内展示 Sprint 1 的最小可用闭环：

- 用户注册并自动进入工作台
- 用户输入教育任务
- 系统识别任务意图
- 系统展示 Agent 阶段状态
- 系统返回结构化摘要和题目结果

## 2. 演示前准备

先在仓库根目录执行：

```powershell
.\scripts\dev-up.ps1
.\scripts\dev-check.ps1
```

确认服务地址：

- 前端：`http://127.0.0.1:5173`
- 后端：`http://127.0.0.1:8000`

## 3. 演示流程

### 场景 1：注册并进入工作台

1. 打开登录页
2. 点击“没有账号，去注册”
3. 输入一个新用户名、手机号、密码
4. 点击“注册”

讲解点：

- 注册成功后会自动登录
- 页面会自动跳转到对话工作台
- 左侧显示当前用户、任务状态、Agent 阶段面板

### 场景 2：使用预设模板

1. 在对话页点击“总结 + 出题”模板
2. 说明模板会自动填充输入框，降低用户输入成本
3. 在模板文本后补一小段学习材料
4. 点击“发送任务”

建议材料：

```text
请总结这段学习材料，并生成 5 个选择题帮助我复习：工业革命推动了机器生产的发展，改变了社会分工与城市化进程。
```

讲解点：

- 左侧状态从“任务提交”进入“意图分析”“工具执行”“结果整理”
- 执行计划会显示识别结果和工具调用顺序
- 右侧结果区不是一段原始文本，而是结构化卡片

### 场景 3：展示不同任务类型

1. 点击“生成复习提纲”模板
2. 直接发送任务

讲解点：

- 当前 Sprint 1 中，这类任务会归入摘要类处理
- 系统仍然会给出稳定结果，而不是报错或空白

### 场景 4：展示异常处理

1. 清空输入框
2. 点击“发送任务”

讲解点：

- 页面不会崩溃
- 会显示“请输入至少 2 个字符的教育任务描述”
- 异常在前端被及时拦截

## 4. 演示时建议说明

- Sprint 1 重点是打通 MVP 主链路，不追求复杂自治
- 当前 Agent 是 LangChain-ready 结构，但工具调用仍是可控的 Mock Tool
- Sprint 2 和 Sprint 3 会继续补过程展示、参数化控制、文件上传、多轮记忆等能力

## 5. 演示结束

演示结束后可以执行：

```powershell
.\scripts\dev-down.ps1
```
